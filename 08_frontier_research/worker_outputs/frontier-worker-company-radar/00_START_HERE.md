# Fresh Parallel Frontier Research OS — Start Here

This folder is a from-scratch Claude Code workflow for parallel deep research into frontier startup ideas in:

1. Power / energy / grid / data-center infrastructure
2. Semiconductors / electronics / advanced packaging
3. Biomedical / medtech / bio-instrumentation
4. Industrial / robotics / advanced manufacturing
5. Extreme systems / cryogenic / superconducting / high-field / harsh-environment technologies

The goal is not to regenerate conservative procurement-ready ideas. The goal is to find **frontier flagship** ideas: products that change a system boundary through extreme power density, torque density, heat flux, magnetic field, reliability, autonomy, miniaturization, precision, or performance.

## Core operating rule

Parallelize discovery. Centralize judgment. Never skip source audit.

## What to run first

After installation and Git setup, start Claude Code from the project root:

```bash
claude --model opus --effort xhigh
```

Then paste:

```text
Execute prompts/00_orchestrator_parallel_frontier_plan.md exactly. Use CLAUDE.md and the source policies in 01_sources/. Do not rank ideas yet. Build the parallel execution plan and confirm the worker output contract.
```

## How this differs from the old workflow

This workflow starts fresh. It does not use your previous 300-source corpus, prior RIDs, or old scoring table. It requires every worker to gather fresh sources and every merge/scoring step to pass the stricter source-audit gate.
